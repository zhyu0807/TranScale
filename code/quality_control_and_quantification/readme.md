1. RNA_bam.smk
Description:
This Snakemake workflow automates the core sequence alignment and post-alignment quality control (QC) steps for transcriptomics data. It ensures that raw sequencing reads are accurately mapped to the reference genome and that the resulting alignment files meet the necessary quality standards for downstream quantification.
Key Processes:
Sample Management: Dynamically loads sample information and file paths (R1 and R2 reads) from a configuration file (bam_config.yaml) and a tab-separated sample list.
Sequence Alignment (rule star_align): Utilizes the STAR (Spliced Transcripts Alignment to a Reference) aligner in 2-pass mode to achieve high-precision mapping of cDNA reads to the genome. It simultaneously generates gene-level counts.
BAM Indexing (rule index_bam): Standardizes the naming of alignment files and generates index files (.bai) using samtools to enable fast random access to the genomic data.
Post-alignment QC (rule qualimap): Employs Qualimap RNA-seq QC to evaluate the quality of the alignment. It generates comprehensive reports (HTML/PDF) covering genomic origin, transcript coverage, and strand specificity.
Dependencies:
Python/Snakemake: Workflow orchestration.
STAR: Genomic alignment.
Samtools: BAM file manipulation.
Qualimap: Alignment quality assessment.

2. RNA_qc.smk
Description:
This Snakemake workflow handles the initial stage of transcriptomics data processing, focusing on raw data quality control (QC) and preprocessing. It performs automated adapter trimming, quality filtering, and comprehensive reporting to ensure the integrity of the data before downstream alignment and quantification.
Key Processes:
Sample Metadata Integration: Automatically parses sample names and raw FastQ file paths from a structured metadata file (sample.txt) to manage multi-sample batch processing.
Preprocessing and Trimming (rule fastp): Uses the fastp tool to perform high-speed quality filtering, base correction, and adapter trimming. It generates both cleaned FastQ files and detailed QC reports in HTML and JSON formats.
Quality Assessment (rule fastqc): Executes FastQC on the cleaned reads to evaluate sequencing quality metrics, such as per-base quality scores and GC content. The reports are automatically organized into directories and compressed for efficient storage.
Dependencies:
Python/Snakemake: Workflow management.
fastp: Fast all-in-one preprocessing for FastQ files.
FastQC: Sequencing data quality assessment.

3. bam_config.yaml
Description:
This configuration file serves as the central parameter repository for the STAR alignment and Qualimap QC workflow (RNA_bam.smk). It defines the paths to essential reference materials and specifies the computational resources and tool-specific parameters required for reproducible data processing.
Key Components:
Sample Metadata Path: Points to the cleaned sample list (clean_sample.txt) containing the metadata for the processed batch.
Reference Genomic Data: Specifies the absolute paths to the merged reference genome (FASTA) and the corresponding gene annotation file (GTF) used for mapping.
STAR Indexing Parameters: Defines the directory for pre-built STAR indices and sets the overhang parameter (e.g., 149) based on the sequencing read length to optimize splice junction mapping.
Resource Allocation:
Memory: Allocates specific Java heap memory (e.g., 32G) for Qualimap to handle large BAM files efficiently.
Parallelization: Configures the number of CPU threads for both genome indexing (16 threads) and read alignment (8 threads) to accelerate the workflow.

4. featur_count.py
Description:
This Python script is designed to aggregate individual gene quantification files generated by featureCounts into a single, unified expression matrix. It streamlines the transition from sample-level quantification to a study-wide dataset, facilitating downstream differential expression or absolute quantification analyses.
Key Processes:
Batch File Processing: Automatically identifies and retrieves all .counts.txt files within a specified directory using pattern matching.
Metadata Extraction: Dynamically extracts sample identifiers from filenames to label the corresponding columns in the final matrix.
Data Integration: Performs an outer merge of individual sample dataframes based on Gene IDs (Ensembl), ensuring that all genes present across the cohort are captured.
Data Cleaning: Handles missing values by filling them with zeros and standardizes the count data into integer format for compatibility with standard bioinformatics tools.
Structured Output: Exports the consolidated data into a standardized CSV format (FeatureCounts.gene_quant.csv), providing a clear "Ensembl-by-Sample" quantification table.

5. featurecount_scale.sh
Description:
This Bash script automates the gene-level quantification process by running featureCounts on all aligned BAM files in a directory. It is specifically configured to handle paired-end RNA-seq data and provides high-resolution counting by accounting for multi-mapping reads and chimeric fragments.
Key Processes:
Batch Processing: Iterates through all sorted BAM files within the alignment directory (05_alignment) to perform systematic quantification across the entire sample set. 
Paired-end Fragment Counting: Utilizes the -p and --countReadPairs flags to ensure that fragments (read pairs), rather than individual reads, are counted as single units of expression. 
Strand-specific Quantification: Configured with -s 2 to match reversely stranded library preparations, ensuring accurate sense/antisense transcript assignment. 
Advanced Read Handling:
Multi-mapping (-M --fraction): Counts multi-mapping reads and assigns them fractional counts to provide a more comprehensive view of repetitive or homologous genomic regions. 
Chimeric & Structural Integrity (-J -B -C): Includes counts for reads mapping to different chromosomes, requires both ends of a pair to be successfully aligned, and excludes chimeric fragments to maintain high data stringency. 
Parallel Execution: Leverages multi-threading (-T 8) to accelerate the quantification of large-scale genomic datasets. 
Dependencies:
Subread package (featureCounts): For genomic feature quantification. 
Bash Shell: For workflow automation.

6. make_filepath.py
Description:
This Python utility script automates the generation of a sample metadata list by recursively searching a specified directory for paired-end sequencing files (.fq.gz). It creates a standardized tab-separated file that serves as the essential input for downstream Snakemake workflows (RNA_qc.smk and RNA_bam.smk).
Key Processes:
Recursive Directory Traversal: Uses os.walk to systematically search through all subfolders within a target directory to locate sequencing data.
Automated Path Resolution: Converts relative file paths into absolute paths, ensuring that the generated sample list remains valid regardless of the working directory during pipeline execution.
Paired-end Recognition:
Automatically identifies and pairs Read 1 (_1.fq.gz) and Read 2 (_2.fq.gz) files based on common filename prefixes.
Extracts the core sample name by stripping the sequencing suffixes.
Structured Metadata Export:
Aggregates information into a Pandas DataFrame with a standardized header: Sample, Sample_path_q1, and Sample_path_q2.
Exports the results as a tab-separated values (TSV) file, ready for direct use in the analysis pipeline.
Command-line Interface (CLI): Implements argparse to allow users to specify input directories and output destinations flexibly via the terminal.
Dependencies:
Pandas: For data structuring and table formatting.
OS/Argparse: For file system navigation and command-line argument handling.

7. result.py
Description:
This Python script is responsible for consolidating individual quantification results generated by RSEM (RNA-Seq by Expectation-Maximization) into comprehensive multi-sample expression matrices. It organizes data into user-friendly Excel formats, facilitating a clear overview of different normalization metrics across the entire study.
Key Processes:
Automated File Discovery: Scans the designated RSEM output directory to identify all .genes.results files, ensuring they are sorted and processed in a consistent order.
Sample Name Extraction: Dynamically parses filenames to extract meaningful sample identifiers by stripping alignment-specific suffixes.
Multi-Metric Consolidation: Simultaneously processes and aggregates three core quantification metrics for every gene across all samples:
Expected Counts: Raw-like counts estimated by the EM algorithm.
TPM (Transcripts Per Million): Within-sample normalized values suitable for comparing relative gene expression levels.
FPKM (Fragments Per Kilobase of transcript per Million mapped reads): Traditional normalization for sequencing depth and gene length.
Structured Data Export: Transforms the consolidated DataFrames into standardized Excel workbooks (.xlsx), creating separate files for Counts, TPM, and FPKM for ease of downstream statistical analysis and visualization.
Dependencies:
Pandas: For advanced data framing and multi-sheet Excel exporting.
Glob/OS/Re: For flexible file pattern matching and string manipulation.

8. rsem.sh
Description:
This Bash script executes the RSEM (RNA-Seq by Expectation-Maximization) pipeline to perform isoform and gene-level expression quantification. It integrates the STAR aligner internally to map reads and utilizes an Expectation-Maximization algorithm to handle read mapping uncertainties, providing highly accurate abundance estimates.
Key Processes:
Integrated Alignment and Quantification: Uses the --star flag to perform read mapping and quantification in a single command, ensuring seamless data flow from FastQ to expression values.
Paired-end & Gzip Support: Configured to handle compressed paired-end sequencing data (--paired-end, --star-gzipped-read-file) directly to save disk space and processing time.
Strand-specific Analysis: Set to --strandedness reverse to correctly process reversely stranded RNA-seq libraries, ensuring accurate transcript-level assignment.
Abundance Modeling: Employs --estimate-rspd to estimate the read start position distribution, which improves the accuracy of expression estimates by accounting for non-uniformity in read coverage.
Output Enrichment:
Generates genome-coordinate BAM files (--star-output-genome-bam) for visualization.
Uses --append-names to include gene and transcript names alongside IDs in the final output files for better interpretability.
High-Performance Computing: Optimized with multi-threading (--num-threads 16) for rapid batch processing of multiple samples.
